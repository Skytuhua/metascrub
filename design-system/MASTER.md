## Design System: MetaScrub

> Canonical, binding source of truth for the build. Baseline generated by the
> design-intelligence engine, then enriched with `--domain` (style/color/
> typography/ux/landing) and `--stack react` deep-dives. Page-specific overrides
> in `design-system/pages/<page>.md` win where they differ; otherwise MASTER
> governs.

### Pattern
- **Name:** Minimal Single Column
- **Conversion Focus:** Single CTA focus. Large typography. Lots of whitespace. No nav clutter. Mobile-first.
- **CTA Placement:** Center, large CTA button (the dropzone *is* the hero CTA).
- **Color Strategy:** Minimalist dark — brand navy + accent green, one accent only.
- **Section order (idle/landing):** 1. Hero headline, 2. Short description, 3. Benefit bullets (3 max: *no upload*, *lossless*, *batch*), 4. CTA (dropzone), 5. Footer.

### Style
- **Name:** Flat Design + Dark Mode (OLED-leaning)
- **Mode Support:** Dark = primary/default ✓ Full | Light = supported ✓ Full (toggle)
- **Keywords:** 2D, minimalist, no shadows/gradients, clean lines, simple shapes, typography-focused, icon-heavy (SVG), high-contrast, eye-friendly dark.
- **Performance:** ⚡ Excellent | **Accessibility:** ✓ WCAG AAA target
- **Technical:** `box-shadow: none` (or at most a hairline border for elevation), `border-radius: 8–12px`, solid fills (no gradients), 1–2px strokes, simplified SVG icons, `color-scheme: dark`.

### Typography
- **Heading:** Lexend (designed for reading proficiency — excellent accessibility)
- **Body:** Source Sans 3
- **Mono:** ui-monospace / system mono — for raw metadata values & coordinates.
- **Mood:** corporate, trustworthy, accessible, readable, professional, clean.
- **Tailwind config:** `fontFamily: { heading: ['Lexend','sans-serif'], body: ['"Source Sans 3"','sans-serif'], mono: ['ui-monospace','SFMono-Regular','monospace'] }`
- **PRIVACY-CRITICAL DELIVERY NOTE:** Do **NOT** load these via the Google Fonts
  CDN `@import` — that is an outbound network request and violates the
  no-data-leaves-the-device promise and the strict CSP (`connect-src 'none'`).
  **Self-host** the woff2 files and declare them with `@font-face` served from
  `'self'`. Provide a system-font fallback stack so the app is fully functional
  even before/without the bundled fonts.

### Key Effects
- No gradients, no drop-shadows (use hairline `--color-border` for separation).
- Simple hover = subtle background/opacity/border-color shift; transitions
  **150–200ms ease**.
- Dark-first: avoid pure-white large fills; text contrast 7:1 where feasible.
- Minimal, purposeful motion only. A short success "scrubbed" check animation is
  fine; **no decorative infinite animation**.
- Loading/async = skeleton or spinner with a label (never a frozen, blank UI).

### Avoid (Anti-patterns)
- **Cluttered data** — metadata is dense; group it, lead with the *sensitive*
  fields, allow raw detail to be collapsed. Don't dump an unstructured wall.
- **Poor credibility** — a privacy tool must *look* trustworthy: no ads, no
  dark patterns, no fake urgency; state plainly how it works.
- **Emojis as icons** — use SVG icons (Lucide/Heroicons-style), never emoji.
- **Frozen UI during async** (parsing/zipping) — always show feedback (UX:Animation/Loading States, High).
- **Continuous/decorative animation** — infinite spins/bounces on non-loaders are distracting (UX:Animation, Medium). `animate-spin` only on actual loaders.
- **z-index whack-a-mole** — understand stacking contexts; isolate with a
  positioned parent rather than `z-index: 9999` (UX:Layout/Stacking Context, Medium).

### Stack Guidelines (React)
- **Don't use `useEffect` for derived state / event handling** — compute during
  render (`const sensitive = fields.filter(...)`), handle events directly. (High — react.dev/learn/you-might-not-need-an-effect)
- Let React **auto-batch** related state updates; avoid `flushSync`. (Low)
- **Profile before optimizing** (React DevTools Profiler); don't guess. (Medium)
- Keep the pure metadata/scrub logic in `src/core/**` (no React/DOM imports) so
  it's directly unit-testable and reusable — UI only orchestrates it.
- Heavy/optional work (HEIC WASM decoder) must be **lazy-loaded** (`import()`),
  never in the initial bundle.

### Pre-Delivery Checklist
- [ ] No emojis as icons (use SVG: Lucide/Heroicons style)
- [ ] `cursor-pointer` on all clickable elements; press/active feedback present
- [ ] Hover + focus + disabled states on every interactive element
- [ ] Visible keyboard focus ring (accent green) for all focusable controls
- [ ] Text contrast ≥ 4.5:1 (target 7:1) in **both** dark and light mode
- [ ] `prefers-reduced-motion` respected (disable non-essential transitions)
- [ ] Responsive at 375 / 768 / 1024 / 1440 px
- [ ] Async ops (parse, scrub, zip) show loading feedback; no frozen UI
- [ ] Fonts self-hosted (no external CDN request); CSP `connect-src 'none'` holds
